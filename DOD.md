# Обновлённый DoD — прототип за 2 часа (single-file architecture)

## 1. Что должно работать к концу 2 часов

Прототип готов, если выполнены **все 7 пунктов**:

| # | Функция | Критерий приёмки |
|---|---------|------------------|
| 1 | **Однофайловая архитектура** | Весь код в `index.html` (HTML + Tailwind + JS + sql.js). Открывается двойным кликом, работает без сервера. |
| 2 | **SQLite через sql.js** | База инициализируется при загрузке. Таблицы: `students`, `errors`, `copy_log`. CRUD работает через sql.js API. |
| 3 | **Двойное хранилище** | SQLite сохраняется в IndexedDB (основное) + localStorage (кэш последней сессии). При перезагрузке страницы данные восстанавливаются автоматически. |
| 4 | **Импорт CSV** | Кнопка «Загрузить CSV» → FileReader парсит файл → данные вставляются в SQLite через транзакцию. Валидация колонок. |
| 5 | **Дашборд с 3 блоками** | SQL-запросы агрегируют данные: «Красный флаг» (GROUP BY student_id, error_type HAVING count ≥ 2), «Факты» (список учеников), «Прогресс» (ошибки, отсутствующие последние 14 дней). |
| 6 | **Карточка ученика + шаблон** | Клик → SQL-запрос по student_id → 2–3 примера ошибок. Текст собирается по шаблону из полей БД (без LLM). |
| 7 | **Копирование + счётчик** | Кнопка «Скопировать» → `navigator.clipboard.writeText()` + INSERT в `copy_log` (student_id, timestamp). Счётчик отображается на дашборде. |

---

## 2. Технические требования (архитектура)

```
index.html (единственный файл)
├── <head>
│   ├── Tailwind CSS (CDN)
│   ├── Font Awesome (CDN)
│   ├── Google Fonts (CDN)
│   └── sql.js (CDN, WebAssembly)
├── <body>
│   ├── Дашборд (3 блока)
│   ├── Карточка ученика (модальное окно)
│   └── Кнопки: Загрузить CSV, Экспорт .db, Экспорт CSV
└── <script>
    ├── Инициализация sql.js + загрузка БД из IndexedDB
    ├── Функции: initDB(), importCSV(), renderDashboard(), showStudent(), copyText()
    ├── SQL-запросы: SELECT, INSERT, GROUP BY
    └── Сохранение в IndexedDB после каждого изменения


## 3. Стоп-метрика (без изменений)

- `copy_count` — общее количество копирований
- `unique_teachers_copied` — уникальные учителя (пока = 1, т.к. один файл)

**Стоп-условие**: за 2 недели пилота `copy_count = 0` → гипотеза не подтверждена.

---

## 4. Что сознательно НЕ делаем

- ❌ Бэкенд, сервер, авторизация
- ❌ LLM / генерация текста
- ❌ Несколько файлов (JS/CSS вынесены)
- ❌ Мобильная адаптация
- ❌ Интеграция с ЭЖ
- ❌ Графики, аналитика, прогнозы
- ❌ Экспорт в облако (только локальный .db)

---

## 5. Чек-лист готовности (последние 15 минут)

- [ ] `index.html` открывается двойным кликом, sql.js инициализируется без ошибок в консоли
- [ ] Загружается демо-CSV (5–7 учеников, 20–30 ошибок) → данные в SQLite
- [ ] Перезагрузка страницы → данные восстановлены из IndexedDB
- [ ] Дашборд показывает ≥1 ученика в «Красном флаге»
- [ ] Клик по фамилии → карточка с примерами ошибок
- [ ] Кнопка «Скопировать» → текст в буфере + запись в `copy_log`
- [ ] Кнопка «Экспорт .db» → скачивается файл базы
- [ ] Счётчик копирований отображается на дашборде
- [ ] Подготовлен скрипт 3-минутного демо

---

## 6. Распределение 2 часов (обновлено под sql.js)

| Блок | Время |
|------|-------|
| HTML-скелет + Tailwind + подключение CDN | 15 мин |
| Инициализация sql.js + создание таблиц + сохранение в IndexedDB | 25 мин |
| Импорт CSV (FileReader + INSERT в SQLite) | 20 мин |
| Дашборд (SQL-запросы + рендер 3 блоков) | 35 мин |
| Карточка ученика + шаблон текста | 20 мин |
| Кнопка «Скопировать» + счётчик + логирование | 10 мин |
| Экспорт .db (Blob + download) | 5 мин |
| Тест на чужой машине + буфер на баги | 10 мин |

**Риск**: sql.js требует асинхронной инициализации WebAssembly. Если возникнут проблемы — откатываемся к чистому localStorage (план Б), но сохраняем структуру данных для будущего миграции.

---

**Главный принцип**: один файл, SQLite в браузере, всё работает офлайн. Мы строим не продукт, а *инструмент измерения* одной метрики — факта копирования текста родителю.
